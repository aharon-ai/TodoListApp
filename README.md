Eine performante Desktop-Anwendung zur Aufgabenverwaltung, die **JavaFX** für das User Interface und **Supabase** als Cloud-Backend nutzt.  
Die Daten werden über eine **REST-API** mit einer PostgreSQL-Datenbank synchronisiert.

## ✨ Features

- **Vollständige CRUD-Unterstützung**  
  Aufgaben erstellen, anzeigen, aktualisieren und löschen.

- **Cloud-Synchronisation**  
  Alle Daten werden sicher in einer PostgreSQL-Datenbank auf Supabase gespeichert.

- **Modernes UI**  
  Responsives Design mit JavaFX-Komponenten und individuellem CSS-Styling.

- **Status-Management**  
  Aufgaben können per Checkbox als erledigt markiert werden.

- **JSON-Integration**  
  Effiziente Datenverarbeitung mit der Google Gson Library.

## 🛠 Installation & Setup

### 1️⃣ Voraussetzungen

- **JDK 17** oder höher
- **Maven**
- Ein **Supabase-Konto**

---

### 🗄 Datenbank-Setup

Erstelle im SQL-Editor deines Supabase-Dashboards die folgende Tabelle:

```sql
CREATE TABLE todos (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  title TEXT NOT NULL,
  description TEXT,
  deadline DATE DEFAULT NOW(),
  done BOOLEAN DEFAULT FALSE
);
```

## 🔐 Row Level Security (RLS)
###  ⚠️ Wichtig
Damit die App Daten speichern und ändern kann, müssen in Supabase unter

`Authentication → Policies`

Regeln erstellt werden.

Erstelle Policies für:

- `SELECT`
- `INSERT`
- `UPDATE`
- `DELETE`

Ohne diese Policies erhältst du beim Speichern einen **403 (Forbidden)** Fehler.

## ⚙️ Konfiguration der Umgebungsvariablen

Die Anwendung nutzt **Umgebungsvariablen**, damit API-Keys nicht im Code gespeichert werden.

Diese Variablen müssen in deiner **IDE Run Configuration** oder im **System** gesetzt werden.

| Variable | Format | Herkunft |
|--------|--------|--------|
| `SUPABASE_URL` | `https://xyz.supabase.co/rest/v1/` | Supabase → Project Settings → API |
| `SUPABASE_KEY` | `eyJhbGci...` | Supabase → Project Settings → API → Anon Key |

## 📖 Beschreibung der Komponenten

### 🔌 SupabaseClient
Verwaltet die Kommunikation mit der **Supabase REST API**  
und übernimmt alle Requests zum **Lesen, Speichern, Aktualisieren und Löschen** der Daten.

---

### 🎮 TodoController
Verbindet die **UI-Elemente** mit der **Anwendungslogik** und den Daten aus dem Backend.

---

### 🖼 FXML
Definiert die **Benutzeroberfläche der Anwendung**.  
Hier werden Layout, Komponenten und Struktur des Interfaces festgelegt.

---

### 🎨 CSS
Enthält das **Styling der JavaFX Oberfläche**  
(z. B. Farben, Abstände, Schriftarten und UI-Design).

---

### ⚙️ module-info.java
Konfiguration des **Java-Modulsystems**.

Ermöglicht außerdem **Reflection für die Gson Library**, damit JSON-Objekte korrekt serialisiert und deserialisiert werden können.
